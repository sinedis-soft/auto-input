# Unified Apps

Папка содержит первые две объединяющие Python-утилиты для разрозненных проектов автоматизации.

## 1. `chrome_capture_app.py`

Универсальное приложение для снимков активной вкладки Chrome. Оператор вводит имя снимка, приложение сохраняет:

- `.png` — скриншот страницы;
- `.txt` — человекочитаемый список видимых полей, кнопок и ссылок;
- `.json` — структурированные данные DOM для дальнейшей настройки селекторов.

Chrome нужно запустить с remote debugging:

```bat
chrome.exe --remote-debugging-port=9222 --user-data-dir="C:\chrome-capture-profile"
python unified_apps\chrome_capture_app.py
```

## 2. `bitrix_policy_router_app.py`

Единое стартовое окно для сделок Bitrix24 и страховых роботов. В приложении есть обязательные элементы:

- кнопки `Настройки`, `Начать`, `Далее`, `Новый полис`, `Сбросить сценарий`;
- поле `ID DEAL BITRIX или ссылка`, куда можно вставлять через Ctrl+V / Paste;
- область `Ход работы`;
- область `Вводимые данные из Bitrix24`, где текст можно выделить и скопировать;
- область `Журнал`, где текст можно выделить и скопировать.

Сейчас маршрутизатор запускает существующие сырые модули:

- WARTA — `warta_robot_app_v2/app.py`;
- ASKO — `asko_bitrix_filler/asko_bitrix_filler.py`.

Маршрутизация выполняется по полям сделки Bitrix24:

- `UF_CRM_1686683031442` — страховая компания;
- `UF_CRM_1690539097` — продукт страхования;
- `UF_CRM_1700656576088` — страна полиса.

ASKO выбирается при компании `1091 / АО «Страховая Компания «АСКО»`, продукте `425 / ОСГО ВТС нерезидента` и стране `1101 / КАЗАХСТАН`. WARTA выбирается при компании `231 / TUiR WARTA S.A.` и продукте `425 / ОСГО ВТС нерезидента`.

Новые страховые сценарии добавляются через список `SCENARIOS` в `bitrix_policy_router_app.py`: нужно указать обязательные поля `required_fields`, ключевые слова fallback-маршрутизации и путь к Python entry point.

Запуск:

```bat
python unified_apps\bitrix_policy_router_app.py
```
