ASKO ← Bitrix24 filler

Назначение:
Приложение открывает Chrome на https://asko2.novelty.kz/login.html, входит в ASKO, забирает сделку Bitrix24 по ID или ссылке и вставляет основные поля в форму нового полиса ОГПО.

ID сделки можно вводить так:
80519
https://sinedis.bitrix24.pl/crm/deal/details/80519/

Настройки хранятся локально:
%USERPROFILE%\.asko_bitrix_filler\settings.json

Важно: в этой версии логин, пароль и webhook хранятся в обычном JSON. Для боевого режима лучше заменить на Windows Credential Manager/keyring.

Запуск:
pip install -r requirements.txt
python asko_bitrix_filler.py

Сборка exe:
build_exe.bat

Готовый файл после сборки:
dist\ASKO_Bitrix_Filler.exe

Что заполняется сейчас:
- Номер бланка ← UF_CRM_1694177619522
- Дата начала ← UF_CRM_1686152149204 / UF_CRM_1693569516501 / BEGINDATE
- Срок ← настройка приложения
- Тип оплаты ← настройка приложения
- Порядок оплаты ← настройка приложения
- Email
- Язык уведомлений
- Форма клиента

Что пока не автоматизировано:
- Страхователь
- Список застрахованных
- Список ТС
- Расчёт премии
- Сохранение

Для этих блоков нужны отдельные сканы окон после нажатия соответствующих кнопок.

- Вставляет этот ИД в поле ASKO ext-comp-1740.
- Ждёт строку autocomplete с этим ИД и текстом «Выверен», затем выбирает её.
- Нажимает «Применить».
- Телефон и примечание пользователя по-прежнему не заполняются.
